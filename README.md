## Reporting bugs

Use public issues for ordinary functional bugs. Include the affected project
and path, exact reproduction steps, expected versus actual behavior, the
version or commit, and sanitized logs or other evidence.

Do not post secrets, credentials, private data, or vulnerability details in a
public issue. Report suspected security vulnerabilities through GitHub's
[private vulnerability reporting](https://github.com/alphazede/developers/security/advisories/new)
route instead.

## Security

This repository has GitHub secret scanning and push protection enabled. Report
suspected vulnerabilities through GitHub's
[private vulnerability reporting](https://github.com/alphazede/developers/security/advisories/new)
route.
